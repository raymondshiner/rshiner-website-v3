## What I do not delegate to AI

Calling this out early because the rest of this site leans hard the other way. The line isn't moving:

- **Architecture decisions on systems that outlive me.** AI accelerates execution, not judgment of trade-offs.
- **Hiring, performance, mentoring, conflict.** People calls. Always.
- **Security and auth code.** Read line by line, every time.
- **Naming things that live in public.** Variables, products, repos. Taste lives here.
- **The choice of what *not* to build.** Scope creep, premature abstraction, over-engineering — the hardest engineering judgment is restraint, and that's still mine.
- **Problem framing before I touch the keyboard.** If I can't write the spec in one paragraph, I'm not ready to start.

AI is execution speed. It isn't the work.

## What's changed in my stack lately

Snapshot of *change*, not inventory. The full skill list lives on [home](/#about).

**Added**
- Tailwind v4 + `@theme` tokens — first design-token solution I've actually shipped with
- Claude Code subagents (Smith, Jarvis, Friday) as a team, not single shots
- Hyprland on Arch (CachyOS) — terminal-first desktop, hand-built bar and popups

**Dropped**
- Cursor and Copilot — the IDE isn't the workshop anymore
- Heavyweight component libraries — shadcn + Radix gets me 90% there with less to fight
- Manual scaffolding for new projects — `smith-init` + a launch script does it in one command

**Watching**
- React 19's compiler in real production code, not toy benchmarks
- MCPs maturing past the demo stage — what's the right unit of composition?
- Whether the "AI-native engineer" label means anything six months from now
